# ADR-0003: Kotlinをコア言語として採用

**ステータス**: 承認済み  
**日付**: 2025-11-12  
**決定者**: プロジェクトアーキテクト  
**関連ADR**: なし

---

## 決定 (Decision)

**選択**: Kotlinをコア言語として採用（Phase 1-7）

Phase 8でパフォーマンス評価を実施し、必要に応じてGo並列エンジンを追加検討。

---

## コンテキスト (Context)

プロジェクトの価値観（保守性 > パフォーマンス > 開発速度）に最適な言語を選択する必要があります。

---

## トレードオフ分析

### Kotlin vs Go vs TypeScript vs Python

| 観点 | Kotlin | Go | TypeScript | Python |
|------|--------|-----|------------|--------|
| **Null安全性** | ✅✅ 言語レベル | ✅ 明示的チェック | △ 弱い | ❌ なし |
| **型安全性** | ✅✅ 実行時も | ✅ 実行時も | ❌ 実行時なし | ❌ 弱い |
| **並列処理** | ✅ Coroutines | ✅✅ Goroutine | △ Worker Threads | ❌ GIL |
| **保守性** | ✅✅ 高 | ✅ 高 | △ 中 | ❌ 低 |

### 得られるもの

- ✅ **型安全性・Null安全性**: コンパイル時にエラー検出
- ✅ **保守性**: 表現力豊かで読みやすいコード
- ✅ **並列処理**: Coroutinesで並列実行可能
- ✅ **JVMエコシステム**: 成熟したライブラリ

### 失うもの

- ❌ **並列処理のシンプルさ**: Goほどシンプルではない
- ❌ **デプロイ**: JVM必要（単一バイナリではない）

### 判断

保守性を最優先するプロジェクトにKotlinが最適。

---

## 段階的アプローチ

```
Phase 1-7: Kotlin のみ
    ↓
Phase 8: パフォーマンス実測（目標: 10,000ゲーム/分）
    ↓

    未達 → Go並列エンジン追加を検討
```
